Nuget what is it

It is a package management system

What is a package

  • One or more assemblies.
  • A list of references to other packages that the assemblies depend on.
  • There is more to an Nuget packages than this but these are these are the aspects that I found interesting at this point.

Components of the system

  • Repository (This may not be the offical term)
  • Client

Repository

  • Is a library of packages
  • They can be remote servers e.g. nuget.org
  • The can be local folders e.g. C:\Users\williampaulmoore\AppData\Local\NuGet\Cache

Client

  • There is a command line client as well as the Visual Studio version
  • You can:
    • Create a packages
    • Push a package to a reposiotory
    • Restore packages (Get the required packages from a repositorhy)

Client configuration

  • Default - %AppData%\Nuget\Nuget.config
  • Will use the configuration file closest to the where it is run from
  • All configurations are loaded.
  • Closer configurations:
    • Overwrite existing single value settings.
    • Are appended to collections e.g. sources.
    • You can clear existing collection in a configuration.
  • Packages.config is an nuget configuration file.
